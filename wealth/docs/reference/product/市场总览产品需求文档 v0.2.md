## 2. 页面基础定义

### 2.1 页面名称

页面名称统一为：**市场总览**。

不得使用以下名称作为正式页面名：

- 首页；
- 首页 / 市场总览；
- 市场驾驶舱；
- 行情首页；
- 市场温度首页。

这些名称可以作为描述性表达，但不能作为页面正式名称。

### 2.2 页面归属

市场总览属于：**乾坤行情系统**。

它是乾坤行情下的第一个页面，而不是一个独立一级菜单。

推荐信息架构表达：

```text
财势乾坤
├── 乾坤行情
│   ├── 市场总览
│   ├── 板块与榜单行情
│   ├── 指数详情
│   ├── 个股详情
│   ├── 我的自选
│   └── 资讯公告
├── 财势探查
├── 交易助手
├── 交易训练
├── 数据中心
└── 系统设置
```

### 2.3 默认入口说明

市场总览可以作为用户打开财势乾坤后的默认落地页，但导航归属仍然是乾坤行情。

因此，产品表达应区分：

| 概念 | 说明 |
|---|---|
| 默认落地页 | 用户进入系统后默认展示市场总览 |
| 一级系统 | 乾坤行情、财势探查、交易助手、交易训练、数据中心、系统设置 |
| 页面归属 | 市场总览属于乾坤行情 |
| 桌面端导航方式 | 不使用固定 SideNav，采用顶部栏 + 面包屑 + 页面内快捷入口 |

## 3. 页面定位

市场总览是乾坤行情系统的入口页，定位为：

> **A 股市场客观事实总览页。**

它用于帮助用户在打开系统后，快速掌握当日 A 股市场的客观运行状态，包括指数表现、涨跌分布、市场风格、成交额、资金流向、涨跌停结构、连板情况、板块强弱和核心榜单入口。

市场总览不是市场分析结论页，不负责直接输出主观判断型指标。

---

## 4. 页面目标

市场总览需要帮助用户快速回答以下问题：

1. 今天 A 股市场整体表现如何？
2. 主要指数涨跌情况如何？
3. 市场涨跌家数分布如何？
4. 市场涨幅分布、跌幅分布如何？
5. 涨跌中位数说明当日赚钱效应如何？
6. 当前市场风格偏大盘还是小盘？偏权重还是题材？
7. 当日成交总额如何？与历史相比处于什么水平？
8. 大盘资金流向如何？
9. 涨停、跌停、炸板、连板情况如何？
10. 连板高度和连板梯队如何？
11. 哪些行业、概念、地域板块涨幅靠前或跌幅靠前？
12. 哪些板块资金流入明显？
13. 哪些个股进入涨跌幅、成交额、换手、量比异动等榜单？
14. 用户应该从哪里继续下钻查看详情？
15. 用户如何快速进入市场温度与情绪、机会雷达、我的自选、我的持仓、提醒中心、用户设置等相关页面？

---

## 5. 页面非目标

市场总览不承担以下职责：

1. 不直接展示市场温度指数；
2. 不直接展示市场情绪指数；
3. 不直接展示资金面分数；
4. 不直接展示短线风险指数；
5. 不直接给出买卖建议；
6. 不输出“看多/看空/建议买入/建议减仓”等交易结论；
7. 不做完整策略验证；
8. 不承载完整机会雷达；
9. 不承载完整持仓深度分析；
10. 不把所有行情、数据、资讯入口堆在一个页面中；
11. 不在桌面端使用固定左侧 SideNav 占用行情内容宽度。

市场温度、市场情绪、资金面分数、风险指数属于“市场温度与情绪分析页”的 P0 指标体系。市场总览只能提供该页面的快捷入口，不展示这些指标的具体分数或分析结论。

## 6. 用户角色与场景

### 6.1 进阶主动投资者

典型场景：开盘前、盘中、收盘后快速查看市场状态。

核心诉求：

- 快速知道今天市场强弱；
- 快速知道赚钱效应是否存在；
- 快速知道资金和板块集中在哪里；
- 快速找到值得进一步查看的指数、板块或个股。

### 6.2 正在建立交易体系的交易者

典型场景：盘中观察市场环境，决定是否进入机会雷达或持仓分析。

核心诉求：

- 判断当日市场环境是否适合继续寻找机会；
- 从客观事实进入财势探查；
- 从榜单和板块下钻到个股；
- 将发现的标的加入自选或提醒。

### 6.3 重视复盘的专业型用户

典型场景：收盘后复盘当日市场结构。

核心诉求：

- 查看当日指数、涨跌分布、成交额、资金、涨跌停结构；
- 对比近半年历史曲线；
- 将市场事实作为后续复盘和策略验证的输入。

## 8. 导航与页面框架要求

### 8.1 桌面端导航原则

市场总览桌面端不采用固定 SideNav。原因：

1. 行情页面需要最大化横向空间，用于表格、图表、板块分布和榜单；
2. 固定 SideNav 会压缩行情内容区，降低金融终端的信息密度；
3. 市场总览作为乾坤行情内的页面，更适合通过面包屑表达页面层级；
4. 高频跳转通过顶部入口、页面内快捷入口和模块下钻完成。

### 8.2 推荐页面框架

桌面端推荐使用：**顶部全局栏 + 面包屑 + 页面标题区 + 内容区 + 页面内快捷入口**。

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ TopMarketBar：品牌 / 全局系统入口 / 主要指数条 / 时间 / 数据状态 / 用户 │
├──────────────────────────────────────────────────────────────────────────┤
│ Breadcrumb：财势乾坤 / 乾坤行情 / 市场总览                              │
│ PageHeader：市场总览 / A股 / 交易日 / 更新时间 / 刷新                    │
├──────────────────────────────────────────────────────────────────────────┤
│ 页面内快捷入口：市场温度与情绪｜机会雷达｜我的自选｜我的持仓｜提醒中心   │
├──────────────────────────────────────────────────────────────────────────┤
│ 首屏核心概览：指数 / 涨跌分布 / 市场风格 / 成交额 / 资金 / 涨跌停       │
├──────────────────────────────────────────────────────────────────────────┤
│ 涨跌停与连板结构                                                        │
├──────────────────────────────────────────────────────────────────────────┤
│ 板块速览 + 榜单速览                                                     │
└──────────────────────────────────────────────────────────────────────────┘
```

### 8.3 TopMarketBar 要求

TopMarketBar 用于承载全局入口和实时市场状态，但不能变成大型导航侧栏。

建议包含：

- 产品标识：财势乾坤；
- 全局系统入口：乾坤行情、财势探查、交易助手、交易训练、数据中心、系统设置；
- 当前主要指数简要行情，可横向滚动或紧凑展示；
- 当前时间；
- 开闭市状态；
- 数据更新时间或数据状态；
- 用户入口。

### 8.4 Breadcrumb 要求

Breadcrumb 是市场总览的主要层级表达方式。

固定表达：

```text
财势乾坤 / 乾坤行情 / 市场总览
```

交互要求：

- 点击“财势乾坤”可返回默认落地页或产品总览；
- 点击“乾坤行情”可展开乾坤行情下的页面列表，或跳转到市场总览；
- 当前页面“市场总览”不可点击或仅作为当前态；
- Breadcrumb 不应占用过高垂直空间。

### 8.5 页面内快捷入口要求

由于不使用 SideNav，页面内需要设置紧凑快捷入口，承接 P0 闭环跳转。

必须包含：

- 市场温度与情绪；
- 机会雷达；
- 我的自选；
- 我的持仓；
- 提醒中心；
- 用户设置。

快捷入口应是轻量模块，不得喧宾夺主，不得挤压首屏核心行情事实。

## 11. 详细功能需求

### 11.1 顶部全局栏

#### 11.1.1 目标

在不占用横向内容区宽度的前提下，提供全局系统入口、关键指数行情、交易时间状态和用户入口。

#### 11.1.2 必须展示

- 产品标识：财势乾坤；
- 一级系统入口：乾坤行情、财势探查、交易助手、交易训练、数据中心、系统设置；
- 主要指数简要行情：上证指数、深证成指、创业板指等；
- 当前时间；
- 开闭市状态；
- 数据更新时间或数据状态；
- 用户头像/用户菜单。

#### 11.1.3 交互

- 点击一级系统入口可展开对应系统页面列表，或跳转至该系统默认页；
- 点击指数行情条可进入对应指数详情页；
- 点击数据状态可查看数据源状态或异常说明；
- 用户入口可进入用户设置。

---

### 11.2 面包屑与页面头部

#### 11.2.1 目标

明确当前页面位置，并展示当前市场、交易日和数据状态。

#### 11.2.2 必须展示

- 面包屑：财势乾坤 / 乾坤行情 / 市场总览；
- 页面标题：市场总览；
- 市场范围：A 股；
- 当前交易日；
- 当前时间；
- 开闭市状态：未开盘、集合竞价、交易中、午间休市、已收盘、非交易日；
- 数据更新时间；
- 数据状态：正常、延迟、部分缺失、异常；
- 手动刷新按钮。

#### 11.2.3 交互

- 点击“乾坤行情”可展开乾坤行情页面列表；
- 点击数据状态，展示数据说明或异常提示；
- 点击刷新按钮，触发局部刷新；
- 非交易日展示最近一个交易日的数据提示。

---

### 11.3 页面内快捷入口

#### 11.3.1 目标

由于不使用固定 SideNav，页面内快捷入口承担 P0 闭环分流职责。

#### 11.3.2 必须提供

- 市场温度与情绪；
- 机会雷达；
- 我的自选；
- 我的持仓；
- 提醒中心；
- 用户设置。

#### 11.3.3 展示规则

- 可以展示入口名称、简短说明、待处理数量或状态；
- 可以展示是否有待处理提醒；
- 可以展示是否已配置自选或持仓；
- 不展示市场温度、情绪指数、资金面分数、风险指数的具体数值。

允许：

```text
市场温度与情绪：查看市场综合分析
机会雷达：查看今日机会线索
我的持仓：查看持仓表现
提醒中心：3 条待处理提醒
```

不允许：

```text
市场温度 82 分
情绪指数进入亢奋区
风险指数提示减仓
```

---

### 11.4 今日市场客观总结

#### 11.4.1 目标

用简短文字总结当日市场事实，但不输出主观投资结论。

#### 11.4.2 文案规则

允许表达：

- 今日主要指数多数上涨/下跌；
- 上涨家数多于下跌家数；
- 成交额较上一交易日放大/缩小；
- 涨停家数增加/减少；
- 连板高度为 N 板；
- 资金净流入/净流出。

禁止表达：

- 市场适合买入；
- 明天大概率上涨；
- 当前风险很低可以加仓；
- 情绪处于冰点/高潮；
- 市场温度达到多少分；
- 短线风险指数提示买卖。

## 14. API 依赖草案

市场总览建议由一个聚合接口支撑首屏加载，同时保留模块接口用于局部刷新。

### 14.1 推荐聚合接口

```text
GET /api/market/home-overview
```

用途：一次性返回市场总览页面首屏和主要模块所需数据。

推荐返回模块：

- tradingDay；
- dataStatus；
- topMarketBar；
- breadcrumb；
- quickEntries；
- marketSummary；
- indices；
- breadth；
- style；
- turnover；
- moneyFlow；
- limitUp；
- limitUpDistribution；
- streakLadder；
- sectorOverview；
- leaderboards。

### 14.2 推荐模块接口

| API | 用途 |
|---|---|
| `GET /api/index/summary` | 主要指数行情与顶部指数条 |
| `GET /api/market/breadth` | 涨跌分布 |
| `GET /api/market/style` | 市场风格 |
| `GET /api/market/turnover` | 成交额总览 |
| `GET /api/moneyflow/market` | 大盘资金流 |
| `GET /api/limitup/summary` | 涨跌停统计 |
| `GET /api/limitup/streak-ladder` | 连板天梯 |
| `GET /api/sector/top` | 板块速览 |
| `GET /api/leaderboard/stock` | 个股榜单 |
| `GET /api/settings/quick-entry` | 快捷入口配置和状态 |

### 14.3 API 边界要求

- 市场总览 API 不返回市场温度分数作为首页核心展示字段；
- 市场总览 API 不返回市场情绪指数作为首页核心展示字段；
- 市场总览 API 不返回资金面分数作为首页核心展示字段；
- 市场总览 API 不返回风险指数作为首页核心展示字段；
- 如需入口状态，只能返回入口是否可用、是否有更新、跳转路径等信息。

## 17. 状态需求

### 17.1 Loading

- 页面首次加载使用骨架屏；
- 顶部指数条、核心模块、榜单模块允许局部 loading；
- 刷新时尽量保留旧数据，显示刷新状态，不整页闪烁。

### 17.2 Empty

需要区分以下空态：

| 空态 | 说明 | 前端提示 |
|---|---|---|
| 非交易日 | 当前日期不是交易日 | 显示最近一个交易日数据 |
| 数据未生成 | 交易刚开始或数据基座未完成计算 | 提示稍后刷新 |
| 单模块无数据 | 某个模块暂缺 | 局部空态，不影响整页 |
| 无权限 | 用户无权查看个人入口数据 | 展示权限提示，不影响市场总览主内容 |

### 17.3 Error

需要区分以下异常：

- 网络异常；
- 服务异常；
- 数据源不可用；
- 数据延迟；
- 字段缺失；
- 部分模块计算失败。

页面应优先保证主框架和已成功模块可用，避免单个模块异常导致整页不可用。

## 20. 合规与表达要求

市场总览必须保持“客观事实页”的表达边界。

### 20.1 允许表达

- 今日上涨家数多于下跌家数；
- 主要指数多数上涨；
- 成交额较上一交易日放大；
- 涨停家数较上一交易日增加；
- 某行业板块涨幅居前；
- 大盘资金净流出。

### 20.2 不允许表达

- 当前可以买入；
- 明日大概率上涨；
- 市场情绪进入高潮；
- 当前风险指数偏高，建议减仓；
- 市场温度已经进入强机会区；
- 该板块值得重仓。
