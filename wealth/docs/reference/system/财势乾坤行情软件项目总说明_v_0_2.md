## 1. 文档定位

《财势乾坤项目总说明》是整个项目的顶层产品与 UI 纲领，用于统一产品定位、模块边界、页面清单、设计原则、API 设计方向、组件体系、HTML Showcase 要求、Codex 实现边界和版本节奏。

本项目后续所有页面设计、接口设计、交互方案、组件规范、Design Token、HTML Showcase、Codex 实现提示词，都应以本文档为上位约束。

任何新功能在进入设计与实现前，都需要先回答三个问题：

1. 它属于哪个系统边界？
2. 它服务于哪一段投资决策链路？
3. 它是否符合“专业、沉稳、闭环型行情决策系统”的产品定位？

如果一个功能无法进入“观察市场、发现机会、验证判断、制定计划、跟踪提醒、复盘训练”的主链路，则应谨慎进入主版本，避免产品演化为普通的信息堆砌型行情软件。

## 3. 产品一句话定位

**财势乾坤是一套面向进阶投资者与专业交易者的综合行情决策支持系统，围绕投资机会发现、交易计划制定与能力训练，帮助用户看清市场、发现机会、验证判断、持续跟踪并复盘成长。**

它不是单纯的行情展示工具，也不是简单的资讯聚合平台，更不是交易下单入口。

财势乾坤的核心价值在于：

- 把行情、资讯、资金、情绪、技术形态、榜单、宏观数据等分散信息组织进同一条决策链路；
- 帮助用户从“看到市场发生了什么”，进一步走向“知道哪些值得跟踪、如何验证、如何计划、如何提醒、如何复盘”；
- 通过长期使用，让用户逐步形成可复用、可验证、可沉淀的交易认知体系。

---

## 4. 产品核心判断

### 4.1 不做“更全的看盘工具”

市场上已经存在大量成熟行情产品，基础行情、资讯公告、资金榜单、技术指标、公式筛选、交易接入等能力已经相当完整。财势乾坤不应以“比别人多几个页面、多几个指标”为主要方向。

财势乾坤真正要解决的问题是：

- 用户每天面对海量行情与资讯，如何快速识别市场状态？
- 用户看到热点后，如何判断它是否值得继续跟踪？
- 用户形成初步判断后，如何验证这个判断？
- 用户发现机会后，如何沉淀成交易计划和提醒规则？
- 用户完成一次判断后，如何复盘这次判断的质量？
- 用户如何通过训练提升盘感、纪律和交易认知？

因此，财势乾坤的方向不是“行情信息更多”，而是“决策链路更完整”。

### 4.2 行情是入口，决策闭环是主线

财势乾坤的主线流程为：

```text
观察市场 → 发现机会 → 验证判断 → 制定计划 → 跟踪提醒 → 复盘训练
```

对应产品能力为：

```text
乾坤行情 → 财势探查 → 技术验证/多维验证 → 交易助手 → 提醒监控 → 交易训练/复盘
```

页面、接口、组件、数据模型都应围绕这条主线组织，而不是按照传统行情软件的“行情、资讯、工具、我的”简单堆叠。

## 5. P0 一期必须完成范围

P0 是第一期必须完成的任务范围。后续页面设计、API 设计、Showcase 和 Codex 实现，应优先围绕本节推进。

### 5.1 P0 一期目标

第一期目标不是一次性做完整产品，而是完成财势乾坤的核心可用闭环：

```text
市场客观事实总览
  → 板块/榜单/指数/个股行情查看
  → 市场温度与情绪等分析指标查看
  → 机会雷达发现机会
  → 自选/持仓登记与管理
  → 持仓深度分析
  → 提醒中心承接交易计划和监控
```

### 5.2 P0 页面清单

| 系统 | P0 页面 | 一期目标 | 备注 |
|---|---|---|---|
| 首页 | 市场总览 | 一屏看清 A 股当日客观市场事实 | 只展示客观事实，不展示主观分析结论 |
| 乾坤行情 | 板块与榜单行情页 | 查看行业/概念/地域板块、板块热力图、核心榜单 | 承接首页下钻 |
| 乾坤行情 | 指数详情页 | 查看主要指数 K 线、技术指标、成分贡献 | 主要指数优先 |
| 乾坤行情 | 个股详情页 | 查看个股行情、K线、技术指标、资金、资讯公告 | 支持进入提醒和交易计划 |
| 乾坤行情 | 我的自选页 | 自选股添加、删除、分组、筛选、查看最新行情 | 可从机会雷达、个股详情加入 |
| 财势探查 | 市场温度与情绪分析页 | 展示市场温度、情绪指数、资金面分数、风险指数及历史曲线 | P0 指标体系核心页 |
| 财势探查 | 机会雷达页 | 发现强势板块、轮动板块、资金回流板块和机会个股 | 支持同步到自选和提醒中心 |
| 财势探查 | 策略验证页 | 记录历史机会分析，并验证机会信号后续表现 | 用于防止机会评分玄学化 |
| 交易助手 | 我的持仓管理页 | 手动登记持仓、资金总量、持仓比例、历史持仓/交易记录 | 不接真实账户 |
| 交易助手 | 我的持仓页 | 展示持仓总览、持仓比例、总市值、余额、当日盈亏估算 | 依赖用户登记数据和行情数据 |
| 交易助手 | 持仓深度分析页 | 基于持仓和当日市场事实分析风险收益 | 可生成交易计划和提醒 |
| 交易助手 | 提醒中心 | 管理自选/持仓/计划相关提醒 | 支持价格、资金、情绪、技术提醒 |
| 系统设置 | 用户设置 | 管理头像、账号、投资偏好 | 投资偏好影响展示与提醒建议 |

### 5.3 P0 暂不纳入强制完成范围

以下能力虽然重要，但不作为第一期必须完成任务：

| 页面/能力 | 暂定优先级 | 原因 |
|---|---:|---|
| 资讯公告页 | P1 | 可先在个股详情中保留资讯公告模块，不单独建设完整资讯页 |
| 技术形态模拟页 | P1 | 属于特色能力，但依赖 K 线、技术指标、策略验证基础，建议第二阶段细化 |
| 交易计划列表/详情 | P1 | P0 先以提醒中心和持仓深度分析承接计划能力，完整计划体系后续展开 |
| 交易训练系统 | P1/P2 | 属于长期成长层，第一期先不阻塞核心闭环 |
| 数据中心前台 | P1 | 数据管理后台已基本实现，前台数据中心不作为 P0 |
| 系统设置 | P1 | 主题、AI 能力配置等可在用户设置之后补齐 |

## 6. 目标用户

财势乾坤优先面向以下用户。

### 6.1 进阶主动投资者

这类用户已经具备一定看盘经验，能够理解指数、板块、个股、K 线、资金流、市场情绪等基础概念，但常常受限于信息分散、缺乏体系、盯盘消耗大。

他们需要的是更高效的市场总览、更清晰的机会线索、更低成本的跟踪提醒，以及更结构化的复盘工具。

### 6.2 正在建立交易体系的交易者

这类用户希望形成自己的选股、观察、买卖、风控、复盘体系，但判断往往停留在模糊经验层面。

他们需要的是能够把模糊判断转化为结构化计划的系统，包括机会评分、验证路径、交易计划、提醒条件、复盘记录和训练反馈。

### 6.3 重视复盘和训练的专业型用户

这类用户不满足于简单看行情，希望通过双盲训练、历史回放、技术形态复盘、错题沉淀等方式提升交易能力。

他们需要的是长期可积累、可对比、可复盘的训练体系，而不是一次性工具。

## 7. 核心价值主线

财势乾坤的产品价值分为两类：效率价值与成长价值。

### 7.1 效率价值

效率价值来自于减少用户在信息搜索、页面切换、机械盯盘中的重复消耗。

具体体现为：

- 一屏看清市场客观状态；
- 快速识别指数、板块、资金、涨跌分布和涨跌停结构；
- 从市场总览直接下钻到板块、榜单、指数、个股和 K 线；
- 将值得关注的机会一键加入自选、提醒或后续计划；
- 对持仓、自选、计划股进行统一监控；
- 用提醒机制替代高强度盯盘。

### 7.2 成长价值

成长价值来自于帮助用户把交易经验沉淀为可复盘、可验证、可训练的能力。

具体体现为：

- 每一次机会发现都能追踪后续表现；
- 每一个交易计划都能记录假设、触发条件和结果；
- 每一次判断都能复盘正确原因或错误原因；
- 每一次训练都能记录能力变化；
- 用户逐步形成稳定交易认知，而不是依赖临场感觉。

## 8. 五大系统边界

财势乾坤由五个核心系统构成：乾坤行情系统、财势探查系统、交易助手系统、交易训练系统、数据中心系统。

五大系统不是平级堆叠，而是围绕决策链路分工协作。

---

### 8.1 乾坤行情系统（P0）

#### 8.1.1 系统定位

乾坤行情系统是事实呈现层，负责帮助用户看清市场，不承担基于市场事实的深度主观分析职责。

它呈现市场行情、指数、板块、个股、K 线、资讯、公告、资金等基础事实，是用户进入财势乾坤的主要入口。

#### 8.1.2 核心职责

- 展示市场全貌，通过主要指数、大盘资金流向、涨跌分布、市场风格等综合性事实指标衡量市场总体状态；
- 提供今日市场状态的客观统计总结；
- 展示核心指数行情、指数成分股、指数指标信息；
- 展示概念、地域、行业板块详情，以及当日涨跌排名、历史排名；
- 展示涨跌停统计数量、连板强度、成交结构、成交额总量、资金流向；
- 展示近半年的历史对比信息，包括涨跌分布走势曲线、成交额总量曲线；
- 展示个股行情、指数行情，包括日线、周线、月线、1分钟、5分钟、15分钟、30分钟、60分钟、90分钟、120分钟 K 线走势、技术指标、个股新闻公告；
- 为财势探查和交易助手提供下钻入口。

#### 8.1.3 不负责事项

- 不直接给出买卖结论；
- 不承担完整策略回测；
- 不承担训练系统职责；
- 不把所有数据入口堆在首页；
- 不在首页输出市场温度、情绪指数、资金面分数、风险指数等主观分析结果。

#### 8.1.4 典型页面

| 页面 | 优先级 | 说明 |
|---|---:|---|
| 市场总览（首页） | P0 | 一屏看清当日行情总结、主要指数行情、当日市场风格、大盘资金流向、涨跌家数分布、涨幅分布、涨跌家数历史曲线、当日市场成交总额、历史成交总额、涨跌停统计、涨跌停分布、连板天梯。 |
| 板块与榜单行情页 | P0 | 展示行业、概念、地域板块当日涨幅前五和跌幅前五、资金流向、涨跌排名前五成分股、个股涨跌幅榜、个股成交额榜、个股换手榜、个股异动榜、板块热力图。 |
| 个股详情页 | P0 | 展示个股行情、K 线、技术指标、资金流向、K线周期切换、十字光标、个股资讯、公告。 |
| 指数详情页 | P0 | 展示主要指数 K 线走势、周期切换、技术指标和成分贡献。 |
| 我的自选页 | P0 | 展示我的自选股票列表，支持股票基本指标、最新交易日行情、添加、分组、筛选、删除等管理能力。 |
| 资讯公告页 | P1 | 聚合市场新闻、个股公告、研报标题、事件线索。 |

### 8.2 财势探查系统（P0）

#### 8.2.1 系统定位

财势探查系统是机会发现与验证层，是财势乾坤区别于普通行情软件的核心特色。

它基于成交量、资金、价格、技术形态、市场温度、市场情绪、风险指数等多维信息，帮助用户挖掘值得跟踪的机会，并提供进一步验证路径。

#### 8.2.2 核心职责

- 发现市场机会；
- 识别板块强弱与资金集中方向，为用户提供判断信息；
- 识别个股异动与技术形态，为用户提供判断信息；
- 展示当日市场温度、市场情绪指数、资金面分数、短线风险指数，并提供历史曲线；
- 对持仓个股进行深度分析，提供风险收益分析，辅助用户形成交易计划；
- 输出机会线索，而不是直接输出交易指令；
- 将机会线索传递给交易助手进行计划与跟踪。

#### 8.2.3 不负责事项

- 不承诺预测准确；
- 不输出绝对买卖建议；
- 不替代用户交易判断；
- 不把评分做成不可解释的黑箱；
- 不将综合分析结果包装成确定性结论。

#### 8.2.4 典型页面

| 页面 | 优先级 | 说明 |
|---|---:|---|
| 市场温度与情绪分析页 | P0 | 展示当日市场温度指数、市场情绪指数、资金面分数、短线风险指数，以及上述数据的历史信息，并综合研判当前市场的风险和收益水平。 |
| 机会雷达页 | P0 | 识别强势板块、轮动板块、资金回流板块，推导强势板块中的强势个股，对个股情况做出综合机会分析和点评，并展示雷达图。支持资金在指数、板块、个股间流向的历史曲线，支持将机会个股同步至提醒中心。 |
| 策略验证页 | P0 | 记录历史机会分析记录，并对机会信号进行历史表现验证。 |
| 技术形态模拟页 | P1 | 通过知名技术分析方法，对个股 K 线形态进行模拟、标注、推演与验证。 |

### 8.3 交易助手系统（P0）

#### 8.3.1 系统定位

交易助手系统是交易计划、跟踪与提醒的承载层。它承接行情系统和探查系统产生的机会线索，帮助用户把模糊判断转化为明确计划，并通过提醒与监控降低盯盘成本。

#### 8.3.2 核心职责

- 由乾坤行情中的我的自选页、交易助手中的我的持仓页进入管理；
- 支持自选股增删、分组；
- 支持持仓股增加、删除、登记；
- 支持从机会探查结果一键加入自选中的特定分组；
- 创建交易计划卡片，并可自动完成监控提醒配置；
- 针对指定股票设置价格、涨跌幅预警、成交额、资金、技术条件等提醒；
- 跟踪并汇总计划执行过程；
- 为复盘系统提供计划与结果数据。

#### 8.3.3 不负责事项

## 9. 决策闭环流程

财势乾坤的核心体验不应以页面堆叠为中心，而应以投资者工作流为中心。

### 9.1 路径一：从首页事实总览进入行情下钻

```text
进入首页
  ↓
查看市场资金、指数、板块、涨跌分布等基本信息
  ↓
发现某个板块或个股异动
  ↓
进入板块/个股详情进行查看
  ↓
点击板块查看板块成分股列表，或点击个股查看 K 线、技术指标、资讯、公告
  ↓
加入自选或生成交易计划，自动记录交易计划
  ↓
设置提醒条件，绑定交易计划
  ↓
系统持续跟踪
  ↓
触发提醒或定期复盘
  ↓
沉淀为训练样本或复盘记录
```

### 9.2 路径二：从市场温度与情绪进入机会雷达

```text
进入市场温度与情绪分析页
  ↓
查看市场温度与情绪分析信息，通过推荐关注进入机会雷达页
  ↓
在机会雷达页发现异动板块和异动个股
  ↓
进入板块/个股详情进行查看
  ↓
点击板块查看板块成分股列表，或点击个股查看 K 线、技术指标、资讯、公告
  ↓
加入自选或生成交易计划，自动记录交易计划
  ↓
设置提醒条件，绑定交易计划
  ↓
系统持续跟踪
  ↓
触发提醒或定期复盘
  ↓
沉淀为训练样本或复盘记录
```

### 9.3 路径三：从持仓进入深度分析和提醒

```text
进入交易助手系统的我的持仓页
  ↓
查看我的持仓情况和持仓分析，管理持仓股
  ↓
点击持仓股进行持仓深度分析，并依据分析结果创建交易计划和交易提醒
  ↓
进入交易计划页查看交易计划表和交易提醒配置
  ↓
第三方平台上收到提醒，点击进入查看该持仓股股票当前行情
  ↓
确认执行交易计划
  ↓
记录交易行为
  ↓
纳入定期复盘
  ↓
沉淀为训练样本或复盘记录
```

### 9.4 页面联动原则

- 首页不是终点，而是市场客观状态入口；
- 情绪、资金、板块之间必须可下钻；
- 个股详情页必须能新增、设置或查看已经设置的提醒和交易计划；
- 机会探查结果必须能进入自选、跟踪与提醒；
- 交易计划最终必须能进入复盘；
- 训练系统应逐步吸收真实交易计划与历史样本。

## 12. 首页定位

首页是财势乾坤最重要的页面之一，它不应是欢迎页，也不应是简单导航页，而应是“市场客观事实驾驶舱”，同时为整个行情系统提供分流入口。

首页只负责展示当日市场客观事实，包括主要指数行情、当日市场风格、大盘资金流向、涨跌家数分布、涨幅分布、涨跌家数历史曲线、当日市场成交总额、历史成交总额、涨跌停统计、涨跌停分布、连板天梯等。

首页不直接展示市场温度、市场情绪指数、资金面分数、短线风险指数等主观分析型结果。这些内容统一进入“市场温度与情绪分析页”。

### 12.1 首页要回答的问题

用户打开软件时，首页应快速回答：

1. 今天市场整体强弱如何，风格如何？
2. 市场涨跌情况如何，分布如何？
3. 整体资金情况如何？
4. 资金主要流向哪些方向？
5. 涨跌停情况如何，连板情况如何？
6. 哪些板块、指数、榜单、个股值得继续下钻？
7. 如何快速进入市场温度与情绪页、我的持仓、我的自选、机会雷达、设置页？

### 12.2 首页核心模块

| 模块 | 作用 |
|---|---|
| 主要指数 | 展示上证、深成、创业板、科创等核心指数当日情况 |
| 涨跌分布 | 展示当日上涨/下跌/平盘家数、历史上涨/下跌/平盘家数、赚钱效应、涨跌中位数、市场风格及历史情况 |
| 涨跌停 | 展示当日连板、炸板、跌停、天地板、地天板以及历史情况 |
| 资金流向 | 展示当日大盘资金流向及历史流向 |
| 板块速览 | 展示当日强势板块、弱势板块、资金流入板块 |
| 榜单速览 | 展示涨跌幅榜、成交额榜、换手榜、量比异动榜入口 |
| 快捷入口 | 进入市场温度与情绪、我的持仓、我的自选、机会雷达、设置页 |

### 12.3 首页设计要求

- 信息密度要高，但层级必须清楚；
- 首屏优先呈现市场客观状态概览，让用户直观感受市场当天强弱；
- 红涨绿跌为硬规则；
- 不使用浮夸大屏风，不做过度动效；
- 首页中的数据模块必须支持下钻；
- 分析型结论通过快捷入口进入探查系统，不在首页混杂展示。

## 14. 指标体系原则

财势乾坤可以使用市场温度、市场情绪、资金面分数、风险指数、机会评分等综合指标，但必须遵守以下原则。

### 14.1 可解释原则

任何综合分数都必须说明：

- 使用了哪些基础字段；
- 各字段如何归一化；
- 权重如何设置；
- 分数区间如何解释；
- 不同状态对应什么含义；
- 该指标不能说明什么。

### 14.2 可验证原则

关键指标应支持后续回测或历史对照。例如：

- 今天市场温度高，次日指数表现如何？
- 情绪分数升温后，短线赚钱效应是否延续？
- 某类机会评分超过阈值后，后续 1 日、3 日、5 日收益如何？
- 资金面改善是否领先板块表现？
- 风险指数升高后，次日跌停、炸板、亏钱效应是否上升？

### 14.3 不神化原则

任何分数都不是买卖结论。页面表达应使用：

- 偏强；
- 偏弱；
- 修复；
- 分歧；
- 退潮；
- 过热；
- 风险升高；
- 值得观察；
- 需要验证。

避免使用：

- 必涨；
- 必跌；
- 马上买；
- 绝佳买点；
- 稳赚；
- 无风险。

## 15. API 设计原则

### 15.1 API 与页面绑定

所有 API 设计优先服务具体页面，不先抽象一套庞大接口体系。

每个页面 API 草案必须包含：

- endpoint；
- method；
- request params；
- response JSON；
- 字段说明；
- 异常状态；
- 前端使用场景；
- 暂缺数据字段清单；
- 性能评估。

### 15.2 API 分层建议

建议 API 按以下方向组织：

```text
/api/market/*          市场总览相关，提供宏观视角下的行情事实信息，包括涨跌分布、涨跌幅分布、市场风格、成交总额
/api/limitup/*         个股涨跌停统计、涨跌停分布、连板天梯
/api/quote/*           个股行情、K线、盘口、技术指标
/api/index/*           指数行情、K线、盘口、技术指标、成分
/api/sector/*          板块行情、板块资金、板块成分，热力图数据
/api/sentiment/*       市场情绪、市场温度、风险指数
/api/moneyflow/*       资金流向
/api/opportunity/*     机会探查、机会评分
/api/watchlist/*       自选股
/api/positionlist/*    持仓股
/api/trade-plan/*      交易计划
/api/alert/*           提醒规则
/api/training/*        训练系统
/api/data-center/*     数据目录、指标字典、数据源状态
/api/settings/*        设置相关，包括系统设置、用户设置
```

### 15.3 响应结构建议

统一响应结构建议：

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "traceId": "req_20260428_000001",
  "serverTime": "2026-04-28T09:30:00+08:00"
}
```

列表响应建议：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [],
    "page": 1,
    "pageSize": 20,
    "total": 100
  },
  "traceId": "req_20260428_000002",
  "serverTime": "2026-04-28T09:30:00+08:00"
}
```

### 15.4 错误码建议

| code | 含义 | 前端处理 |
|---:|---|---|
| 0 | 成功 | 正常渲染 |
| 400001 | 参数错误 | 提示用户检查筛选条件 |
| 401001 | 未登录 | 跳转登录或提示授权 |
| 403001 | 无权限 | 展示权限提示 |
| 404001 | 数据不存在 | 展示空状态 |
| 409001 | 状态冲突 | 提示刷新后重试 |
| 429001 | 请求过快 | 限流提示 |
| 500001 | 服务异常 | 展示异常状态和重试按钮 |
| 503001 | 数据源不可用 | 展示数据源异常提示 |

## 16. UI 视觉原则

财势乾坤的视觉目标是：**专业、沉稳、高密度但不杂乱，有金融终端感。**

### 16.1 风格关键词

- 专业；
- 沉稳；
- 克制；
- 清晰；
- 高信息密度；
- 金融终端感；
- 数据驱动；
- 可长期使用。

### 16.2 禁止方向

避免以下视觉方向：

- 低幼卡通风；
- 娱乐化游戏风；
- 廉价大屏风；
- 过度霓虹发光；
- 大面积无意义渐变；
- 花哨装饰压过数据；
- 用红绿之外的随意颜色表达涨跌；
- 用“科技蓝大屏”削弱金融产品识别度。

### 16.3 涨跌颜色硬规则

中国股票市场默认：

- 红色代表上涨；
- 绿色代表下跌；
- 灰色代表平盘或无变化；
- 黄色/金色可用于品牌强调、机会提示、重点标记，但不得与涨跌方向冲突。

此规则适用于：

- 指数涨跌；
- 个股涨跌；
- K 线涨跌；
- 涨跌幅文字；
- 榜单颜色；
- 图表标注；
- 热力图；
- Tooltip；
- HTML Showcase mock 数据。

### 16.4 信息密度原则

- 表格、图表、卡片是核心表达单元；
- 重要数据优先，不做空洞大卡片；
- 信息分组要清晰，避免满屏无层级；
- 首屏承载核心市场信息，提供客观事实；
- 次级信息通过折叠、Tab、Drawer 或详情页承载。

## 19. 交互设计原则

### 19.1 下钻优先

行情软件的交互重点不是弹窗堆砌，而是清晰下钻。

典型路径：

```text
首页 Top 板块 → 板块详情
首页 Top 个股 → 个股详情页 → K线分析 → 加入自选/提醒/计划
```

### 19.2 所见即行动

用户看到一个值得关注的信息时，页面应提供自然动作：

- 加入自选；
- 加入观察；
- 创建计划；
- 设置提醒；
- 查看详情；
- 对比同板块；
- 查看历史表现。

### 19.3 解释优先

对于综合分数、机会信号、情绪状态，必须提供解释入口：

- 为什么是这个分数？
- 由哪些指标构成？
- 哪些指标贡献最大？
- 当前状态与历史相比处于什么位置？
- 有什么风险？

### 19.4 状态明确

交易计划、提醒、训练、复盘都应有明确状态。

例如交易计划状态：

```text
草稿 → 观察中 → 等待触发 → 已触发 → 执行中 → 已结束 → 已复盘
```

## 20. 空状态、加载态、异常态原则

### 20.1 空状态

空状态不能只显示“暂无数据”，应告诉用户原因与下一步动作。

示例：

- 当前筛选条件下没有机会信号，可放宽涨幅、成交额或板块条件；
- 当前股票暂无资金流数据，可稍后刷新或查看基础行情；
- 尚未创建交易计划，可从个股详情或机会雷达中创建。

### 20.2 加载态

行情类页面应优先使用骨架屏和局部加载，避免整页空白。

- 表格加载显示骨架行；
- 图表加载显示网格占位；
- 指标卡加载显示数字占位；
- 数据刷新时不应导致页面剧烈跳动。

### 20.3 异常态

异常态应区分：

- 网络异常；
- 服务异常；
- 数据源延迟；
- 交易日无数据；
- 权限不足；
- 参数错误；
- 指标计算失败。

每类异常应提供对应处理动作。

## 21. HTML Showcase 规范

HTML Showcase 是给 Codex 和前端工程师还原页面的重要依据，应尽量接近高保真原型。

### 21.1 基本要求

- 使用单文件 HTML/CSS/JS；
- 不依赖复杂构建工具；
- 可以使用 CSS、SVG、Canvas 模拟图表；
- 页面中必须使用真实感 mock 数据；
- 所有涨跌颜色必须遵守红涨绿跌；
- 页面视觉要符合专业、沉稳、金融终端感；
- 不能做成廉价大屏风；
- 所有核心交互要在 JS 中有基础模拟；
- 组件要使用与 Design System 和 Design Token 一致的表达。

### 21.2 Showcase 必须表达

每个页面 Showcase 至少表达：

- 页面整体布局；
- 顶部/侧边导航关系；
- 核心模块位置；
- 模块间关系和距离；
- 组件间距离和分隔；
- 数据密度；
- 关键交互；
- hover/active/selected 状态；
- loading/empty/error 的视觉方向；
- mock 数据结构。

### 21.3 K 线类 Showcase 特殊要求

K 线页面必须表达：

- 周期切换；
- K 线主图区；
- 均线；
- 成交量；
- MACD；
- KDJ；
- 十字光标；
- 主图和指标区横坐标联动；
- 右侧个股基本指标区；
- 点击股票列表进入详情页的路径。

## 24. 暂不做事项

## 25. 合规与表达边界

财势乾坤应定位为行情分析、决策辅助、训练复盘工具，而不是投资顾问或收益承诺工具。

页面表达应避免：

- 明确指令式买卖建议；
- 保证收益；
- 暗示稳赚；
- 过度夸大预测能力；
- 将综合评分包装为确定性结论。

推荐表达：

- 当前市场偏强/偏弱；
- 当前情绪处于修复/分歧/退潮；
- 该方向值得继续观察；
- 该信号需要结合资金和价格确认；
- 风险正在升高；
- 可加入观察并设置提醒；
- 后续需要复盘验证。

## 29. 总结

财势乾坤的核心方向，不是重复一个更复杂的行情终端，而是围绕投资者真实决策过程，建立一套可观察、可验证、可计划、可跟踪、可复盘、可训练的专业行情决策支持系统。

当前阶段的核心任务，是优先完成 P0 一期闭环：

```text
市场客观事实总览
  → 板块/榜单/指数/个股行情查看
  → 市场温度与情绪分析
  → 机会雷达
  → 自选与持仓管理
  → 持仓深度分析
  → 提醒中心
```

项目总控的核心任务，是持续守住以下原则：

- 产品上，守住“决策闭环”；
- 模块上，守住“五大系统边界”；
- 阶段上，守住“P0 一期必须完成范围”；
- 页面上，守住“用户任务和主线价值”；
- 首页上，守住“客观市场事实入口”；
- 视觉上，守住“专业、沉稳、金融终端感”；
- 行情规则上，守住“中国市场红涨绿跌”；
- API 上，守住“页面数据依赖和可实现性”；
- 版本上，守住“先主线，后扩展”；
- 协作上，守住“文档先行，Showcase 对齐，Codex 按规范实现”。

这份文档后续应作为整个项目的母文档，持续协调产品、UI、前端、API、数据和 Codex 实现工作。
