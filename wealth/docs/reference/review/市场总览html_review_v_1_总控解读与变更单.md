## 1. 本轮 Review 目标

本轮 Review 不推翻市场总览页面方向，而是在保留“市场总览是乾坤行情下的 A 股市场客观事实总览页”这一定位基础上，对 `market-overview-v1.html` 做结构、信息密度、图表表达和交互细节优化。

本轮核心目标：

1. 修复明确 UI 缺陷；
2. 强化红涨绿跌；
3. 减少模块说明文字对页面空间的占用；
4. 将多个装饰性趋势图升级为具备坐标、时间切换和 Tooltip 的可读图表；
5. 优化成交额、资金流、涨跌停、连板天梯等模块的信息结构；
6. 调整市场总览主体布局，使页面更像专业行情终端；
7. 保持市场总览“客观事实页”边界，不引入市场温度、情绪指数、资金面分数、风险指数作为首页核心结论。

## 2. 已确认的改版基线

### 2.1 页面仍然遵守的总原则

1. 页面名称：市场总览；
2. 所属系统：乾坤行情；
3. 不是独立一级菜单；
4. 桌面端不使用固定 SideNav；
5. 使用 TopMarketBar + Breadcrumb + PageHeader + ShortcutBar + 全宽行情内容区；

8. 中国市场红涨绿跌；

## 4. 关键设计变更说明

### 4.1 图表交互统一化

本轮涉及多个历史趋势图，后续应统一为标准图表交互模式：

1. 左上角提供 `1个月 / 3个月` 切换；
2. 横坐标必须显示时间或日期；
3. 纵坐标必须显示数量、百分比或金额单位；
4. 鼠标移动显示坐标定位线；
5. Tooltip 显示日期与当前点数值；
6. 图表应服务数据阅读，不做纯装饰线条。

适用模块：

- 涨跌分布历史趋势图；
- 市场风格历史趋势图；
- 成交额历史趋势图；
- 大盘资金流历史趋势图；
- 涨跌停历史柱状图。

### 4.2 模块说明文字收纳

模块标题下方的解释性文字不再直接占用正文空间。

处理方式：

```text
模块标题 + 圆圈问号图标
```

hover 或点击问号时展示说明 Tooltip。

适用模块：

- 今日市场客观总结；
- 涨跌分布；
- 市场风格；
- 成交额总览；
- 大盘资金流向；
- 涨跌停统计与分布；
- 连板天梯；
- 板块速览；
- 榜单速览。

### 4.3 涨跌停统计与分布合并

原“涨跌停分布列表”废弃。

新模块结构建议：

```text
涨跌停统计与分布
├── 核心统计卡
│   ├── 涨停家数
│   ├── 跌停家数
│   ├── 炸板家数
│   ├── 封板率
│   ├── 最高连板
│   └── 天地板/地天板
├── 当日涨跌停分布图
│   ├── 涨停板块分布
│   ├── 跌停板块分布
│   └── 炸板分布
└── 历史涨跌停组合柱图
    ├── 1个月/3个月切换
    ├── 涨停红色
    ├── 跌停绿色
    └── Tooltip 显示日期、涨停数、跌停数
```

### 4.4 连板天梯横向化

连板天梯保持独立模块，紧跟在涨跌停统计与分布之后。

新结构：

```text
连板天梯
├── 首板（数量）
├── 二板（数量）
├── 三板（数量）
├── 四板（数量）
└── 五板及以上（数量）
```

每个层级内部展示股票卡片。

卡片字段建议：

- 股票名称；
- 股票代码；
- 所属板块；
- 最新价；
- 涨跌幅；
- 开板次数；
- 封单金额，如数据具备；
- 首次封板时间，如数据具备。

### 4.5 板块速览处理

板块速览保持独立一行，并保留当前布局设计。

本轮不对板块速览做结构性重构，只调整它在页面中的整体顺序。
