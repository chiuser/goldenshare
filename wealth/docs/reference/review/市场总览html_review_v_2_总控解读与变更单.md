## 1. 本轮 Review 目标

本轮 Review v2 是对市场总览 Showcase 的局部精修，不是重新设计整页，也不是推翻 Review v1 的方向。

本轮核心目标：

1. 只修改用户在 Review v2 中明确指出的问题区域；
2. 修正“今日市场客观总结 + 主要指数”的顶部布局，使其回到左右结构；
3. 强化“榜单速览”表格的信息完整度，展示 Top10，并补充换手率、量比、成交量、成交额等列；
4. 调整“涨跌停统计与分布”内部 2×2 结构，并补充“昨天涨停板块分布 + 跌停/炸板结构”；

## 5. 详细变更要求

### 5.1 今日市场客观总结 + 主要指数

#### 5.1.1 当前问题

当前版本中，“今日市场客观总结”和“主要指数”被拆成各自独占一整行，导致首屏拉长、内容关系变弱。

#### 5.1.2 修改要求

恢复左右结构：

```text
┌──────────────────────────────┬──────────────────────────────┐
│ 今日市场客观总结              │ 主要指数                      │
│ 50%                           │ 50%                           │
└──────────────────────────────┴──────────────────────────────┘
```

左侧“今日市场客观总结”结构：

```text
今日市场客观总结
├── 5 个事实卡片
└── 说明性文字卡片
```

右侧“主要指数”结构：

```text
主要指数
├── 第一行：5 个指数
└── 第二行：5 个指数
```

#### 5.1.3 禁止事项

1. 不得把“今日市场客观总结”和“主要指数”拆成两个独占整行模块；
2. 不得减少主要指数为单行；
3. 不得把主要指数卡片改成横向滚动；
4. 不得改变未被点名的其它首屏模块。

---

### 5.2 榜单速览

#### 5.2.1 当前问题

当前榜单速览展示内容偏少，字段不足，Top 数量不足，行情终端的信息密度不够。

#### 5.2.2 修改要求

榜单速览表格展示 Top10。

表格列固定为：

```text
排名｜股票｜最新价｜涨跌幅｜换手率｜量比｜成交量｜成交额
```

字段说明：

| 列名 | 说明 | 涨跌色规则 |
|---|---|---|
| 排名 | 当前榜单排名 | 中性色 |
| 股票 | 股票名称 + 股票代码 | 中性色 |
| 最新价 | 股票最新价 | 随涨跌方向变化，上涨红、下跌绿、平盘灰白 |
| 涨跌幅 | 当前涨跌幅 | 上涨红、下跌绿、平盘灰白 |
| 换手率 | 当日换手率 | 中性色，可高值强调但不使用涨跌色 |
| 量比 | 当日量比 | 中性色，可高值强调但不使用涨跌色 |
| 成交量 | 当日成交量 | 中性色 |
| 成交额 | 当日成交额 | 中性色 |

#### 5.2.3 布局要求

- 可相应增加榜单速览板块高度；
- 保持与大盘资金流向同一行、二等分的整体布局关系，除非当前 Showcase 已经被用户确认调整；
- 表格密度要高，但不能牺牲可读性；
- 表格行 hover 状态必须清晰；
- 点击个股进入个股详情。

#### 5.2.4 API 影响

榜单 API 必须返回或确认：

- rank；
- stockCode；
- stockName；
- latestPrice；
- changePct；
- turnoverRate；
- volumeRatio；
- volume；
- amount。

---

### 5.3 涨跌停统计与分布区

#### 5.3.1 当前问题

当前涨跌停统计与分布区域中，历史涨跌停组合柱状图和左下方分布结构的位置不符合用户预期；同时缺少昨天的涨停板块分布与跌停/炸板结构。

#### 5.3.2 修改要求

该区域改为 2×2 布局。

正确结构：

```text
┌──────────────────────────────┬──────────────────────────────┐
│ 8 个统计卡片                  │ 今日涨停板块分布 + 跌停/炸板结构 │
├──────────────────────────────┼──────────────────────────────┤
│ 历史涨跌停组合柱状图          │ 昨天涨停板块分布 + 跌停/炸板结构 │
└──────────────────────────────┴──────────────────────────────┘
```

#### 5.3.3 左上：8 个统计卡片

建议包含：

1. 涨停家数；
2. 跌停家数；
3. 炸板家数；
4. 封板率；
5. 最高连板；
6. 连板家数；
7. 天地板数量；
8. 地天板数量。

如当前 Showcase 已有更合理 8 卡组合，可保持，但必须满足“8 个卡片”的结构。

#### 5.3.4 右上：今日涨停板块分布 + 跌停/炸板结构

展示今日结构。

建议包含：

- 今日涨停板块分布；
- 今日跌停结构；
- 今日炸板结构；
- 可用紧凑图形、分布条、矩阵或小型结构图表达；
- 不使用普通长列表。

#### 5.3.5 左下：历史涨跌停组合柱状图

展示历史趋势。

要求：

- 横坐标是日期；
- 纵坐标是数量；
- 支持 1个月 / 3个月切换；
- 涨停红色；
- 跌停绿色；
- 同一日期柱组中展示涨停与跌停；
- Tooltip 显示日期、涨停数、跌停数。

#### 5.3.6 右下：昨天涨停板块分布 + 跌停/炸板结构

新增昨日结构，用于与今日结构形成对照。

建议包含：

- 昨天涨停板块分布；
- 昨天跌停结构；
- 昨天炸板结构；
- 表达形式与右上今日结构保持一致；
- 必须清晰标明“昨天”或“上一交易日”。

#### 5.3.7 API 影响

API 需要支持：

- 今日涨停板块分布；
- 今日跌停/炸板结构；
- 昨天涨停板块分布；
- 昨天跌停/炸板结构；
- 历史涨停/跌停序列；
- 8 个统计卡片字段。

### 5.4 板块速览

#### 5.4.1 当前问题

当前板块速览没有按用户希望的“多榜单矩阵 + 热力图”方式呈现。

#### 5.4.2 修改要求

板块速览独占一行，内部结构如下：

```text
┌──────────────┬──────────────┬──────────────┬──────────────┬──────────────────┐
│ 行业涨幅前五 │ 概念涨幅前五 │ 地域涨幅前五 │ 资金流入前五 │                  │
│ 1.xxxx       │ 1.xxxx       │ 1.xxxx       │ 1.xxxx       │                  │
│ ...          │ ...          │ ...          │ ...          │  板块热力图       │
├──────────────┼──────────────┼──────────────┼──────────────┤  5 行 × 4 列     │
│ 行业跌幅前五 │ 概念跌幅前五 │ 地域跌幅前五 │ 资金流出前五 │                  │
│ 1.xxxx       │ 1.xxxx       │ 1.xxxx       │ 1.xxxx       │                  │
│ ...          │ ...          │ ...          │ ...          │                  │
└──────────────┴──────────────┴──────────────┴──────────────┴──────────────────┘
```

关键点：

1. 左侧是 4 列 × 2 行榜单矩阵；
2. 右侧板块热力图独立占两行高度；
3. 板块热力图内部为 5 行 × 4 列；
4. 不是只在第一行放热力图；
5. 热力图与上下两排榜单在视觉高度上对齐；
6. 板块速览仍独占一整行。

#### 5.4.3 左侧 8 个榜单块

上排：

```text
行业涨幅前五｜概念涨幅前五｜地域涨幅前五｜资金流入前五
```

下排：

```text
行业跌幅前五｜概念跌幅前五｜地域跌幅前五｜资金流出前五
```

每个榜单块展示 Top5。

每个列表项建议字段：

- 排名；
- 板块名称；
- 涨跌幅或资金净流入/流出；
- 领涨股或代表股，如空间允许。

#### 5.4.4 右侧板块热力图

要求：

- 5 行 × 4 列；
- 跨左侧两行高度；
- 红涨绿跌；
- 可以展示板块名称和涨跌幅；
- hover 显示成交额、资金流、上涨/下跌成分数量等；
- 点击板块进入板块与榜单行情页。

#### 5.4.5 API 影响

API 需要支持：

- 行业涨幅 Top5；
- 行业跌幅 Top5；
- 概念涨幅 Top5；
- 概念跌幅 Top5；
- 地域涨幅 Top5；
- 地域跌幅 Top5；
- 资金流入 Top5；
- 资金流出 Top5；
- 热力图 20 个格子数据。
