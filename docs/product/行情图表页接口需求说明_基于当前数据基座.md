# 行情图表页接口需求说明（基于当前数据基座）

## 1. 文档目的

本文档用于定义 **iOS 股票行情软件 QuoteDetail 图表页** 所需的后端接口。  
目标不是定义全站所有接口，而是优先支撑当前已经在客户端实现中的这几个能力：

- 个股详情页基础信息
- 顶部价格摘要区
- 三层扩展信息区
- 周期切换
- K 线主图
- 成交量 / MACD / KDJ 副图
- 复权切换
- 十字线查看所需字段
- 后续真实接口替换当前 Mock 数据

本文档会同时说明：

1. **建议提供的接口**
2. **每个接口的请求与响应结构**
3. **当前数据基座中可以直接支撑的字段**
4. **需要服务端派生计算的字段**
5. **当前数据基座中明显缺失的字段**

---

## 2. 当前数据基座可直接支撑的核心数据

基于当前数据目录，以下数据集对行情图表页最关键：

### 2.1 证券基础信息
- `stock_basic` -> `core.security`
- `index_basic` -> `core.index_basic`
- `etf_basic` -> `core.etf_basic`

可提供：
- 证券代码
- 名称
- 行业
- 市场
- 交易所
- 上市状态
- 上市日期等

### 2.2 日线行情
- `daily` -> `core.equity_daily_bar`
- `index_daily` -> `core.index_daily_serving`
- `fund_daily` -> `core.fund_daily_bar`

可提供：
- `open/high/low/close`
- `pre_close`
- `change_amount`
- `pct_chg`
- `vol`
- `amount`

### 2.3 日频基础行情指标
- `daily_basic` -> `core.equity_daily_basic`
- `index_daily_basic` -> `core.index_daily_basic`

可提供：
- `turnover_rate`
- `volume_ratio`
- `pe_ttm`
- `total_mv`
- `circ_mv`
- `pb`
- `ps`
- `dv_ratio`
等

### 2.4 周线 / 月线
- `stk_period_bar_week` -> `core.stk_period_bar`
- `stk_period_bar_month` -> `core.stk_period_bar`
- `stk_period_bar_adj_week` -> `core.stk_period_bar_adj`
- `stk_period_bar_adj_month` -> `core.stk_period_bar_adj`
- `index_weekly` -> `core.index_weekly_serving`
- `index_monthly` -> `core.index_monthly_serving`

可提供：
- 周 / 月 K 线
- 周 / 月前复权 / 后复权价格（股票）

### 2.5 复权因子
- `adj_factor` -> `core.equity_adj_factor`

可用于：
- 服务端推导股票日线前复权 / 后复权
- 为客户端提供统一复权 K 线接口

### 2.6 交易日历
- `trade_cal` -> `core.trade_calendar`

可用于：
- 校验交易日
- 周期聚合
- 前端分页边界处理
- 识别非交易日

### 2.7 资金流 / 龙虎榜 / 涨跌停等补充数据
- `moneyflow` -> `core.equity_moneyflow`
- `top_list` -> `core.equity_top_list`
- `limit_list_d` -> `core.equity_limit_list`
- `block_trade` -> `core.equity_block_trade`
- `dividend` -> `core.equity_dividend`
- `stk_holdernumber` -> `core.equity_holder_number`

这些更适合后续扩展，不是当前图表页 MVP 的硬依赖。

## 3. 当前图表页建议采用的接口策略

为了降低客户端复杂度，建议后端先提供两类接口：

### 3.1 页面聚合接口（BFF）
用于一次性返回详情页顶部和外围信息。

### 3.2 图表序列接口
用于返回指定周期、指定复权方式的 K 线及指标数据。

这样做的原因：

- 顶部摘要区与图表区刷新频率不同
- 周期切换时只需要刷新图表序列
- 页面首屏初始化时可减少客户端组装成本
- 后续真实行情接入时，聚合接口和图表接口可以独立演进

---

# 4. 接口总览

建议第一阶段先提供以下 5 个接口：

1. `GET /api/v1/quote/detail/page-init`
2. `GET /api/v1/quote/detail/kline`
3. `GET /api/v1/quote/detail/related-info`
4. `GET /api/v1/quote/detail/announcements`
5. `GET /api/v1/market/trade-calendar`

如果希望客户端更简单，也可以把 3 和 4 聚合进 1。  
但从演进性看，建议保留独立接口。

# 5. 接口 1：详情页初始化接口

## 5.1 接口定义

## 5.2 接口目标

用于返回详情页首屏初始化所需的非大体量数据，包括：

- 三层扩展信息区
- 图表头部默认数值（基于最后一根 candle）
- 默认图表请求参数

## 5.3 请求参数

- `symbol + market`
- 或统一使用 `ts_code`

- `instrument_id`

# 6. 接口 2：图表序列接口（核心）

## 6.1 接口定义

`GET /api/v1/quote/detail/kline`

## 6.2 接口目标

用于返回图表页真正绘图所需的 K 线序列数据。  
这是图表页最核心的接口。

## 6.3 请求参数

| 参数 | 类型 | 必填 | 示例 | 说明 |
|---|---|---:|---|---|
| `symbol` | string | 是 | `600519` | 证券代码 |
| `market` | string | 否 | `SH` | 市场 |
| `security_type` | string | 否 | `stock` | `stock` / `index` / `etf` |
| `period` | string | 是 | `day` | `timeLine` / `minute5` / `minute15` / `minute30` / `minute60` / `day` / `week` / `month` |
| `adjustment` | string | 否 | `forward` | `none` / `forward` / `backward` |
| `start_date` | string | 否 | `2025-01-01` | 查询起始日期 |
| `end_date` | string | 否 | `2026-04-03` | 查询结束日期 |
| `limit` | integer | 否 | `300` | 默认 300，最大建议 2000 |

### 参数约束
- `stock` 支持 `none/forward/backward`
- `index` 和 `etf` 建议仅支持 `none`
- 若请求了不支持的复权方式，后端返回 400 或自动降级为 `none` 并给 warning 字段

## 6.5 服务端职责

服务端必须统一完成以下工作，**不要把这些逻辑扔给客户端**：

1. 根据 `period` 返回相应周期数据
2. 根据 `adjustment` 返回相应复权结果
3. 计算技术指标：
   - MA5 / 10 / 20 / 60 / 120 / 250
   - Volume MA5 / 10
   - MACD
   - KDJ
4. 返回用于十字线和头部数值栏的完整字段
5. 保证 bars 按时间升序返回，最后一根是最新

## 6.6 数据来源建议

### 股票日线
- 基础日线：`core.equity_daily_bar`
- 基础指标：`core.equity_daily_basic`
- 复权因子：`core.equity_adj_factor`

### 股票周/月线
- 不复权：`core.stk_period_bar`
- 前/后复权：`core.stk_period_bar_adj`

### 指数日线
- `core.index_daily_serving`
- 指数基础指标可叠加 `core.index_daily_basic`

### 指数周/月线
- `core.index_weekly_serving`
- `core.index_monthly_serving`

### ETF 日线
- `core.fund_daily_bar`

## 6.7 当前明显缺失

### 缺失 1：分钟级 / 分时级数据
当前数据目录里没有看到用于以下周期的分钟线数据集：

- `timeLine`
- `minute5`
- `minute15`
- `minute30`
- `minute60`

所以这部分**当前数据基座无法直接支撑**。  
要么补分钟级数据表，要么暂时把这些周期标记为“不支持”。  
这一点非常关键。fileciteturn1file1L1-L60 fileciteturn1file3L1-L13

### 缺失 2：ETF 周/月线
目录里有 `fund_daily`，但没看到 ETF 周/月聚合或 ETF 复权周期表。  
如需 ETF 周/月图，需要新增服务端聚合层或新增底表。fileciteturn1file3L1-L13

## 7.1 接口定义

## 7.2 接口目标

```json
      "action_target": "885525.TI"

```

## 7.5 当前可用数据来源

### 行业
- `core.security.industry`

### 概念 / 板块 / 成分
当前目录中有一些可参考数据：
- `core.ths_member`
- `core.dc_member`
- `core.kpl_concept_cons`
- `core.ths_index`
- `core.dc_index`

但这些更像“成分关系”或“板块定义”，要拼成“单个股票 -> 推荐相关概念”还需要服务端做关系整理。fileciteturn1file3L1-L13

## 7.6 当前明显缺失

### 缺失 1：股票 -> 相关 ETF 映射关系
当前只有 `core.etf_basic`，但没有“股票 / 行业 / 概念 -> ETF”的现成映射表。  
所以如果你想保留“相关ETF”这块，需要新增一张映射关系表或推荐规则。fileciteturn1file3L1-L13

### 缺失 2：统一的 symbol-to-concept serving 视图
虽然底表分散存在，但缺少一个面向前端查询的统一 serving 视图。  
建议新增：
- `core.security_related_tag_serving`
或
- `core.security_concept_serving`

# 8. 接口 4：公告接口

## 8.1 接口定义

`GET /api/v1/quote/detail/announcements`

## 8.2 接口目标

返回第二层“公告滚动轮播区”数据。

## 8.3 请求参数

| 参数 | 类型 | 必填 |
|---|---|---:|
| `symbol` | string | 是 |
| `market` | string | 否 |
| `limit` | integer | 否，默认 5 |

## 8.4 返回结构

```json
{
  "items": [
    {
      "id": "ann_20260401_001",
      "title": "2026年第一季度业绩预告已发布",
      "ann_date": "2026-04-01",
      "category": "earnings",
      "source": "announcement_service"
    }
  ]
}
```

## 8.5 当前数据基座情况

**当前上传的数据目录中没有看到公告正文 / 公告列表 / 新闻快讯数据集。**

所以这个接口**当前数据基座无法直接支撑**。  
需要你判断以下方案之一：

### 方案 A：暂时不做真实公告接口
继续前端 mock。

### 方案 B：接外部公告源
例如交易所公告、资讯源等。

### 方案 C：先用低频事件替代部分公告
可以尝试用这些低频事件做“伪公告”：
- `dividend`
- `stk_holdernumber`

但这只能覆盖极少数事件，不能替代真正公告流。fileciteturn1file4L17-L52

# 11. 当前数据基座缺失字段 / 缺失能力清单

下面这部分是最关键的，你可以直接拿去判断要不要补。

## 11.1 分钟级 / 分时级行情
当前未见可直接支撑的数据表：

- 分时
- 5分
- 15分
- 30分
- 60分

### 影响
- 当前图表页的分钟周期无法真实实现

### 建议
- 新增分钟线表
- 或先把分钟周期标记为“暂不支持”

---

## 11.2 公告 / 新闻 / 快讯
当前未见：
- 公告正文表
- 公告列表表
- 快讯资讯表

### 影响
- 公告滚动轮播区无法真实落地

### 建议
- 新接公告源
- 或暂时保留 mock

---

## 11.3 股票与 ETF 的关联映射
当前有 `etf_basic`，但没有：
- 股票 -> ETF 推荐关系
- 行业 / 概念 -> ETF 映射关系

### 影响
- “相关ETF”无法可靠生成

### 建议
- 新增映射表
- 或暂时只返回行业 / 概念，不返回 ETF

---

## 11.4 统一的概念 / 板块 serving 视图
当前概念相关数据分散在：
- `ths_index`
- `ths_member`
- `dc_index`
- `dc_member`
- `kpl_concept_cons`

### 影响
- 前端想拿“当前股票的相关概念 / 板块”时，需要后台再做很多拼装
