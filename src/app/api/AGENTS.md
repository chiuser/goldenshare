# AGENTS.md — `src/app/api` 聚合入口规则

## 适用范围

本文件适用于 `src/app/api/` 及其子目录。

---

## 目录定位

`src/app/api` 是应用入口聚合层，不是业务实现层。

负责：

1. `/api` 与 `/api/v1` 的聚合入口编排
2. 子路由挂载顺序与统一入口组织
3. 轻量健康检查入口组织

不负责：

1. Biz API 主实现
2. Ops API 主实现
3. Auth/Admin 业务规则实现

---

## 硬约束

1. 不在本目录新增业务逻辑主实现。
2. 改入口时保持路径与返回契约兼容。
3. 非必要不调整 `include_router` 顺序。

---

## 改动后说明

每次改动需说明：

1. 变更是否仅限入口聚合
2. 影响了哪些子路由
3. 是否保持 `/api`、`/api/v1/*` 行为稳定
