# AGENTS.md — `src/app` 组合根规则

## 适用范围

本文件适用于 `src/app/` 及其子目录。

---

## 目录定位

`src/app` 是组合根（composition root），负责装配，不承载核心业务规则。

负责：

1. Web 应用创建与运行入口
2. Router 聚合
3. Auth wiring 与依赖注入装配
4. 异常处理装配
5. App 层共享 schema / model registry 接线

不负责：

1. Biz 业务查询/业务规则
2. Ops 运维治理规则
3. Foundation 底层同步与存储规则

---

## 约束

1. 不把 `app` 变成“大杂烩业务层”。
2. `app/api` 只做聚合，不写业务实现。
3. `app/web` 只做运行入口装配，不写业务规则。
4. 涉及行为变更时，先确认接口契约不漂移。

---

## 改动后说明

每次改动需说明：

1. 改动属于装配层哪一类职责
2. 是否引入业务逻辑（必须为否）
3. 是否影响对外入口行为
